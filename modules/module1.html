Module 1: Introduction to IoT Connectivity & Matter

This module introduces why Matter was introduced and how it changes smart-home interoperability.

Learning Goals

  • Understand the evolution of smart home standards.
  • Compare Zigbee, BLE, Wi-Fi, and Matter.
  • Understand why Matter solves interoperability challenges.

Key Concepts

  • Smart home architecture: devices, gateways, controllers, cloud services, and user interfaces.
  • Interoperability: different vendors sometimes use incompatible protocols and ecosystems.
  • Matter: a unified IP-based protocol for smart home devices.
  • CSA ecosystem: Connectivity Standards Alliance manages the Matter ecosystem and certification.

Hands-on Lab

Analyze a sample smart home ecosystem with a light, plug, and sensor from different vendors. Document how they connect and where Matter helps.

  1. List the device types in the ecosystem.
  2. Identify the transport used by each device (Wi-Fi, BLE, Thread, Zigbee).
  3. Explain how Matter enables shared control and onboarding.

Suggested Tools

  • Matter SDK / Connected Home IP
  • Wireshark for traffic analysis
  • Presentation slides and a simple architecture diagram

Quick Demo Code

The following C++ example shows a simple device model representation that can be used in teaching slides or early prototypes.

#include <iostream>
#include <string>
#include <vector>

struct Device {
    std::string name;
    std::string transport;
    std::string role;
};

int main() {
    std::vector<Device> ecosystem = {
        {"Smart Light", "Thread", "Leaf device"},
        {"Smart Plug", "Wi-Fi", "Controller-connected"},
        {"Temperature Sensor", "BLE", "Sensor node"}
    };

    for (const auto& device : ecosystem) {
        std::cout << device.name << " uses " << device.transport
                  << " as " << device.role << "\n";
    }
    return 0;
}
