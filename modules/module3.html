Module 3: Matter Commissioning & Interaction Model

This module covers commissioning flow, QR code onboarding, secure sessions, commands, attributes, and events.

Key Concepts

  • BLE commissioning for initial setup.
  • Device discovery and QR code onboarding.
  • Secure session establishment using certificates and authentication.
  • Commands, attributes, events, and cluster communication.

Hands-on Lab

Commission a Matter smart plug into a controller and perform basic commands such as on/off and status read.

  1. Pair the device using the commissioning flow.
  2. Use CHIP Tool (or equivalent Matter tool) to discover the device.
  3. Issue a command and read a status attribute.

Reference C++ Example

#include <iostream>
#include <string>

struct CommissioningFlow {
    std::string discovery;
    std::string secureSession;
    std::string command;
};

int main() {
    CommissioningFlow flow{"QR code + BLE", "Certificate-based session", "OnOff command"};
    std::cout << "Discovery: " << flow.discovery << "\n";
    std::cout << "Session: " << flow.secureSession << "\n";
    std::cout << "Command: " << flow.command << "\n";
    return 0;
}
