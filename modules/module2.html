Module 2: Matter Architecture & Device Model

This module explains the Matter data model, device topology, and how endpoints and clusters relate to a real device.

Core Concepts

  • Node: a physical or virtual device in the network.
  • Endpoint: a functional part of a device such as a light or a switch.
  • Cluster: a group of attributes, commands, and events.
  • Device Type: defines the behavior of a product category.

Architecture Walkthrough

A Matter lighting system commonly contains a light endpoint with an on/off cluster, a level control cluster, and a color control cluster. The application layer maps these clusters to device behavior.

Reference C++ Example

#include <iostream>
#include <string>
#include <vector>

struct Cluster {
    std::string name;
    std::string purpose;
};

struct Endpoint {
    int id;
    std::string role;
    std::vector<Cluster> clusters;
};

int main() {
    Endpoint light{1, "Lighting", {{"OnOff", "Turn the light on or off"}, {"LevelControl", "Dim the light"}}};
    std::cout << "Endpoint " << light.id << " (" << light.role << ")\n";
    for (const auto& cluster : light.clusters) {
        std::cout << "- " << cluster.name << ": " << cluster.purpose << "\n";
    }
    return 0;
}
